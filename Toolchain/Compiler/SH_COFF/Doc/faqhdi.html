Q1. How do I use GNUSH/H8 ELF tool chain with HDI ?

A1. To debug the programs compiled by using ELF tool chain, ELF/DWARF2 reader is required by HDI. This updated ELF/DWARF2 reader for HDI is available for download from web-based support request system http://www.kpitgnutools.com/renesasTools.php

Please note that this is the most recent version of ELF/DWARF2 reader for HDI than that has been referenced in the FAQ previously. The user should download and upgrade the reader with this version even if this is done before.

User should overwrite this gdwarf2.dll in HDI installation folder. Please ensure dwarf2.dll is removed from the HDI installation directory.

Q1. How do I debug my programs using HDI?

A1.To debug your programs using HDI the following steps should be followed :

1. First you have to pass -g switch to command line for C/C++ files while compiling the application.

2. The target hardware should be flashed with HDI-M. The monitor program communicates with the HDI to debug the user program.

3. Since HDI-M is in control, the user program (the program to be debugged) must be loaded in RAM. If you are using our Sample Tutorials, use 'make ram' to build the application which in turn compiles without -DROMSTART option.

4. Also, all the sections should be located in RAM.

5. To debug the programs compiled by using ELF tool chain, ELF/DWARF2. This updated ELF/DWARF2 reader for HDI is available for download from web-based support request system available at http://www.kpitgnutools.com

6. Some times HDI does not load the *.out file completely. To make sure that data is loaded correctly, load S-Record *.mot file first and then load the corresponding *.outfile. This will ensure that all sections are correctly loaded.

7. Once all the above are taken care of, you will be able to source level debug the user program.

Note: For the debugging procedure of EVB 3644 please refer to the corresponding manual.

Q2. Although I have deselected the -mint32 option, HDI (coff) is handling it as a 32 bits integer. Why?

A2. The GNU compiler exports a different data type for "int" and "short int" as a part of the debug information. This entirely depends upon the source file and how the user specifies the data type. If it contains "int", the h8300-coff compiler emits debug information for "int" in your program (default - 32 bits), which should have been 16 bits for H8 targets. To avoid this confusion between data types we suggest you to use "short int" when you intent to use 16 bit integers. Alternatively you may use h8300-elf tool chain, which does not have this problem.

Q3. I am unable to load the H8 normal mode files in HDI. The HDI reports Incorrect Format. What should I do?

A3. Current HDI object reader does not support Normal mode targets for H8/300H and H8S. You need to update the HDI object reader. The latest HDI Object reader for normal mode targets is available at http://www2.eu.renesas.com/products/mpumcu/tool/hdi/support.html#ubr for the download.

The updated HDI reader is also available for download from our web-based support request system http://www.kpitgnutools.com

Q1. How do I debug my programs using HDI?

A1.To debug your programs using HDI the following steps should be followed :
1. First you have to pass -g switch (-gdwarf-2 in case of ELF tool chain) to the command line for C/C++ files while compiling the application.
2. The target hardware should be flashed with HDI-M. The monitor program communicates with the HDI to debug the user program.
3. Since HDI-M is in control, the user program (the program to be debugged) must be loaded in RAM. If you are using our Sample Tutorials, use 'make ram' to build the application which in turn compiles without -DROMSTART option. All the sections should be located in RAM.
4. You can source level debug assembly files using ELF tool chain by using -gdwarf2 switch while assembling the *.s files.
5. If you are using preprocess assembly files (*.S), you can source level debug assembly files using ELF tool chain by using Wa,-gdwarf2 switch to *-elf-gcc.
For details about Renesas (formerly Hitachi) HDI please visit: http://www2.eu.renesas.com/products/mpumcu/tool/hdi/support.html
