GNUSH v0501

 

 

February 4th, 2005

 

KPIT Cummins Infosystems Limited is now releasing GNUSH v0501, a cross compiler tool chain for Renesas
(formerly Hitachi) SH series of micro controllers.

SALIENT FEATURES:

  1. GNUSH v0501 is based on gcc-3.4.3, binutils-2.15 and newlib-1.12.0.

  2. The latest patches have been applied to gcc, binutils and newlib.

  3. A new code size optimization option '-msort-data' is added to the compiler. Please visit http://www.kpitgnutools.com/optimization_in_GCC.html#msortdata.

NOTE: GDBSH v0402 can be used with GNUSH v0501.

Please refer to the FAQ for details of the patches applied to the gcc-3.4.3 sources.

CHANGES IN THIS RELEASE:

This section summarizes the major changes as compared to GNUSH v0403.

GCC:

1

If variable declared in interrupt routine takes more than 256 bytes, it was causing ICE. This is fixed in this release.

2

The structure offset byte is wrong for the following C-source code with '-mrenesas' option. Without "-mrenesas" option it gives correct offset that is 4.
The data "b2" of structure "bbb" offset is comming 8 which is wrong, it should be 4.
struct aaa{
int a1;
}*zzz;
struct bbb {
int b1:1;
int b2;
};
test(){
zzz = &yyy;
((struct bbb*)zzz)->b2 = 0x12345678;
}
This has been fixed.

Windows and GNU/Linux:

2

Following code gives Internal Compiler Error for targets SH2e, SH3e, SH4, SH4-single, SH4-single-only with -O2 optimization option,
    double bar (int x, int z1, double z2)
    {
     int y;
     if (z1 > z2)
      y = z1 + z2;
     else
      y = 0;
     return ((double) x + y);
    }
Workaround is to pass option -fno-cse-follow-jumps with -O2.

3

Libraries built with -mrenesas option for SH2 & SH3 little endian targets are not working correctly.
